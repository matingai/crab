# Code of Conduct

Crab is an experimental agent runtime project. The community should be useful, direct, and
welcoming to people who are exploring agent systems, local automation, Rust, desktop
runtime design, and developer tooling.

## Expected Behavior

- Be respectful when disagreeing about design, architecture, benchmarks, or model behavior.
- Keep feedback actionable. Prefer examples, reproduction steps, logs, traces, and
  alternatives over vague criticism.
- Assume good intent, but do not use that assumption to dismiss real harm or repeated
  disruption.
- Make room for both deep technical discussion and beginner questions.
- Respect privacy. Do not post API keys, private logs, personal data, or third-party
  confidential material.

## Unacceptable Behavior

- Harassment, threats, discrimination, personal attacks, or sexualized language.
- Publishing another person's private information without explicit permission.
- Repeatedly derailing technical conversations after maintainers ask for focus.
- Submitting malicious code, intentionally misleading reports, or unsafe instructions.
- Pressuring maintainers or contributors to disclose private credentials, accounts, or
  commercial information.

## Enforcement

Maintainers may remove content, close issues, block accounts, or restrict participation
when behavior makes the project unsafe or unproductive. Enforcement should be
proportionate, documented when practical, and focused on protecting the project space.

For sensitive safety or conduct concerns, use GitHub's private reporting features when
available. Do not include secrets or private third-party information in public issues.
